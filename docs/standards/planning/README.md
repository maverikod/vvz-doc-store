# Planning Standards Library

These standards preserve the complete HRS/MRS/GS/TS/AS hierarchy, semantic
reproduction, human-owned binding HRS, typed MRS, cascade, status discipline,
zero-trust re-read, mechanical-before-semantic verification, dependency ordering,
and one-file atomic-step rule. For doc-store, Plan Manager through MCP Proxy is
normative plan truth, Code Analysis Server is first for project search and
analysis, and local `apply_patch` is the only project content mutation route.
Generic `docs/plans/<plan_name>` paths in methodology examples do not make local
plan files authoritative.
