# Native role translation

Executable roles are duty prompts, not model classes or distinct APIs. The root owns
user communication and final aggregation. Ordinary children may be assigned researcher,
reviewer, tester, GS owner, TS owner, AS author/executor, implementation-owner, or
bounded-coder duties. Escalation changes the bounded duty/context, not the model.

Historical source tiers (Fable, Opus, Sonnet, Haiku) are provenance only: respectively
root ownership, high-complexity research/ownership, default research/ownership, and
atomic author/coder duties. They never select or identify a runtime model.

The owner/mini/spark aliases from older execution material map to root owner, TS
context-former/verifier child, and one-AS executor child. Their hierarchy, waiting, and
verification duties remain; any historical model field is non-binding metadata.
